AR_UNDEFINED_INDEX Constant

The AR_UNDEFINED_INDEX constant serves as a sentinel value indicating an undefined or invalid index. Its value is architecture-dependent, corresponding to the maximum possible value representable by the INDEX data type. Since INDEX is an unsigned integer, this constant is typically defined either directly (e.g., ((INDEX) 0xFFFFFFFF)) or by casting a signed -1 to the unsigned type (((INDEX) -1)).

Version:1.0
Header file:AR_Types.h (include AR_API.h)
